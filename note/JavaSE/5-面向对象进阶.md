## 二、继承（Extends）

### 1、介绍

让一个类（子类 / 派生类）去复用另一个类（父类 / 超类）的**非私有成员**，实现代码复用，是面向对象三大特征之一，使用关键字`extends`。

父类：`super class`

子类：`sub class`

Java的继承是单继承，即一个类只能直接继承**一个**父类；也可以实现**多层继承**（A→B→C），`Object`是所有类的父类（这个父类不一定是直接父类）

:warning:Java里面创建（new）子类对象的时候会首先把父类对象创建（new）出来。

按照本节中 `一、UML类图` 中展示的UML类图所展示的关系编写代码：

```java
public class Person {
    protected int id;
    protected String name;
    protected int age;
    protected String gender;

    public void show() {
        System.out.println("Person.show");
    }
}

public class Student extends Person{
    private String className;

    public void study() {
        System.out.println("Student.study");
    }
}

public class Teacher extends Person{
    private double salary;

    public void teach() {
        System.out.println("Teacher.teach");
    }
}
```

### 2、继承中的this和super

**this：**代表当前类的对象 `this.id=id`

**super：**代表父类对象，在子类中调用父类的属性或方法

**this()：**调用当前类的其他构造方法

**super()：**调用父类构造方法

需要注意的是，在子类中写一个无参构造，当不写任何代码时，默认添加一个`super()`来调用父类的无参构造。

## 三、重载（Overload）

同一个类中，`方法名`相同，`参数列表`不同，多个方法互为重载（Overload）。

Java通过方法的`参数类型`、`个数`或`顺序`来区分不同的重载方法。

**最典型的重载：**构造方法重载（无参构造方法和有参构造方法，方法名相同，但参数列表不同）

---

## 四、重写（Override）

**重写（Override）**发生在父子类（继承关系）之间
子类定义一个方法，方法签名和父类被继承的方法完全一致，子类覆盖父类方法实现，称为重写，重写方法前要加上`@Override`注解.

满足条件：存在继承（extends）或实现（implements）关系，方法名、参数列表必须和父类一模一样。

>- 子类覆盖父类，必须保证子类权限大于父类权限才可以覆盖，否则会编译失败。
>
>- 如果父类是`private`的方法，子类是默认方法，那么子类是不知道这个方法的，不是覆盖。

```java
class Animal{
    public Animal getObj(){return null;}
}
class Dog extends Animal{
    @Override
    public Dog getObj(){return null;}
}
```

## 五、多态（Polymorphism）

### 1、介绍

**Java多态（Polymorphism）**是面向对象编程的一项核心概念，它允许对象以`多种形式`出现。

面向对象三大特征：封装、继承、**多态**。

>多态存在的三个必要条件：
>
>- 继承
>
>- 重写
>
>- 父类引用指向子类对象（向上转型）
>
>  ```java
>  Animal animal = new Dog(); // 向上转型，多态写法
>  animal.say(); // 执行子类重写的方法
>  ```

![6dbd8b3d-e467-4f1c-938a-4a2960ffecb3](./../assets/5-%E9%9D%A2%E5%90%91%E5%AF%B9%E8%B1%A1%E8%BF%9B%E9%98%B6/6dbd8b3d-e467-4f1c-938a-4a2960ffecb3.png)

```java
public class Cal {
    protected int num1;
    protected int num2;

    public Cal(int num1, int num2){
        this.num1 = num1;
        this.num2 = num2;
    }

    public int getResult(){
        return 0;
    }

}

public class Add extends Cal{

    public Add(int num1, int num2){
        super(num1, num2);
    }

    @Override
    public int getResult(){
        return num1 + num2;
    }
}

public class Sub extends Cal{
    public Sub(int num1, int num2){
        super(num1, num2);
    }

    @Override
    public int getResult() {
        return num1 - num2;
    }
}

public class Mul extends Cal{
    public Mul(int num1, int num2){
        super(num1, num2);
    }
    @Override
    public int getResult(){
        return num1 * num2;
    }
}

public class Div extends Cal{
    public Div(int num1, int num2) {
        super(num1, num2);
    }
    @Override
    public int getResult() {
        return num1 / num2;
    }
}
```

实现多态：

```java
@Test
public void test2() {
    Cal cal = new Add(3,2);
    System.out.println(cal.getResult());

    cal = new Sub(3,2);
    System.out.println(cal.getResult());
}
```

输出：

```bash
5
1
```

### 2、编译时多态和运行时多态

多态又分为`编译时多态（静态多态）`和`运行时多态（动态多态）`，Java真正意义上的多态是`运行时多态`。

**编译时多态**，又称静态多态，对应的是`方法重载（Overload）`；

**运行时多态**，又称动态多态，对应的是`方法重写（Override）`，依托`向上转型`实现多态。

| 对比项       | 编译时多态（静态多态） | 运行时多态（动态多态） |
| ------------ | ---------------------- | ---------------------- |
| 实现方式     | 方法重载 Overload      | 方法重写 Override      |
| 发生时机     | 编译期绑定             | 运行期绑定             |
| 绑定类型     | 静态绑定 / 早绑定      | 动态绑定 / 晚绑定      |
| 位置         | 同一个类               | 父子继承体系           |
| 是否需要继承 | 不需要                 | 必须要有继承 / 实现    |
| 核心判断依据 | 参数列表               | 对象实际类型           |

## 六、抽象方法和抽象类（Abstract）

上述章节中的`Cal`类中的`getResult`方法没有任何意义，我们可以将`Cal`类定义为**抽象类**，将其中的`getResult`定义为抽象方法。

### 1、抽象方法

使用 `abstract` 修饰，**只有方法声明，没有方法体**（没有 `{}`，以分号结尾）

- 抽象方法**不能有方法体**

- 包含抽象方法的类，**必须声明为抽象类**

- 构造方法、静态方法、private 方法**不能定义为 abstract**

>原因：
>
>- static 静态方法属于类，无法被子类重写；
>- private 私有方法子类不可见，无法重写；
>- 构造方法不能被继承重写。

### 2、抽象类

**抽象类不能 new 创建对象，无法实例化**

可以包含：

1. 普通成员变量

2. 静态变量

3. 构造方法（作用：供子类创建对象时 `super()` 调用）

4. 普通实例方法

5. 静态方法

6. **抽象方法（可选！抽象类可以没有抽象方法）**

> :warning:重点：抽象类**不一定**有抽象方法；但是有抽象方法的类**一定**是抽象类。

抽象方法可以理解为父类（抽象类）定义的一个模板或者规范（自己本身不实现）；

子类继承抽象类（`extends`）的时候，普通子类**必须重写父类所有抽象方法**；如果子类不重写全部抽象方法，子类也要定义成 `abstract` 抽象类。

---

回到这一小节最开始的那个问题，将Cal定义为抽象类，将其中的getResuit方法定义为抽象方法：

```java
public abstract class Cal {
    protected int num1;
    protected int num2;

    public abstract int getResult();
}
```

这样写也能避免新增加的子类忘记重写getResult方法，因为普通子类**必须重写父类所有抽象方法**或也定义成抽象类。

>当方法的参数类型声明成父类类型，那么所有的子类对象都可以传递进来：
>
>```java
>    @Test
>    public void test3(){
>        Cal cal = new Add(5,2);
>        jisuan(cal);
>    }
>
>    public void jisuan(Cal cal){
>        System.out.println(cal.getResult());
>    }
>```

## 七、接口（Interface）

### 1、介绍

接口是**行为规范 / 契约**，使用关键字 `interface` 定义，他没有提供任何是实现,，具体的功能由实现接口的子类去实现。

接口是一个**纯粹的抽象类**（接口里面**所有的方法都是抽象方法**）。

定义方法：

```java
interface 接口名 {
}
```

类实现接口使用 `implements`：

```java
class Dog implements Run{
}
```

### 2、面向接口编程

**接口分为广义和狭义：**

1. 狭义：就是Java里面接口这个技术 interface
2. 广义：声明成父类类型，指向不同的子类对象。

这里的父类可以是：接口、抽象类、普通父类；**接口也是父类的一种**

```java
//接口就是一个纯粹的抽象类，接口就是一个标准
public interface IFly {
    //public abstract void fly();
    void fly();
}
```

### 3、接口的特点

接口和抽象类一样，都不能实例化，且接口**没有构造方法**，这也是接口和类最大的区别。

访问权限：

- 接口内抽象方法、默认方法**隐式 public**，不能写 private/protected
- 接口本身只能用 `public` 或默认包访问

接口是一个纯粹的**抽象类**，接口里面的方法都是抽象方法，会被默认加上 `public abstract` ，不用单独声明；

接口中的所有变量都是静态的常量，所有变量都会被默认加上`public static final`，也不用单独声明。

```java
double PI = 3.1415926535;//等同于public static final PI = 3.1415926535;
public static final PI = 3.1415926535;
```

### 4、选择接口和抽象类

**继承**是严格的**父子关系**，你继承什么你就是什么，**比如：鸽子就是鸟。**

父类是抽取了所有子类的共同特征，父类里面的属性和方法是所有子类所共有的属性和方法。

**接口**是一个**标准**，一般用在能不能具有**某个功能**。

把能不能飞定义称为一个接口IFly，实现了这个接口的类就具有了这个功能。

![1efb90ed-1632-4969-911a-ab345daa7123](./../assets/5-%E9%9D%A2%E5%90%91%E5%AF%B9%E8%B1%A1%E8%BF%9B%E9%98%B6/1efb90ed-1632-4969-911a-ab345daa7123.png)

```java
public abstract class AbstractBird {
    abstract void egg();
}

public interface IFly {
    void fly();
}

public class DaYan extends AbstractBird implements IFly {
    @Override
    void egg() {
        System.out.println("DaYan egg");
    }

   @Override
    public void fly() {
        System.out.println("DaYan fly");
    }
}

public void test() {
    DaYan daYan = new DaYan();
    daYan.egg();
    daYan.fly();

    // bird是站在AbstractBird这个类角度去看new出来的DaYan这个对象，
    // 所以只能看到egg()这个方法
    AbstractBird bird = new DaYan();
    bird.egg();
    // bird.fly();调用不到
    // bird.show();调用不到

    // 站在能不能飞IFly的角度去看DaYan，只能看到fly()这个方法
    IFly fly = new DaYan();
    fly.fly();
    //fly.egg();调用不到
    //fly.show();调用不到
    fly = new GeZi();
    fly.fly();
}
```

### 5、final用法总结

final是最终的意思，用于修饰**变量、类、方法**，用法总结如下：

**final修饰变量：**变量不能被修改，即为常量；

```java
public static final double PI  = 3.1415;
```

**final修饰类：**这个类不能被继承，没有子类；

**final修饰方法：**这个方法不能被重写。

## 八、静态属性和静态方法

### 1、介绍

被 `static` 修饰的成员，属于**类本身**，不属于对象实例。

### 2、静态属性

#### 定义

```java
class Student{
    // 静态属性
    public static String school = "青岛农业大学";
    // 实例属性
    public String name;
}
```

#### 特点

- 静态属性属于类，不属于对象
- 在类加载的时候就初始化，早于对象创建
- 所有对象共享同一份静态属性，即修改一处，全部对象看到的值都会变化
- 生命周期跟随类，类卸载之后才会释放

### 3、静态方法

#### 定义

```java
public static void showSchool(){
    System.out.println(school);
}
```

#### 特点

静态方法归属类，不属于对象

静态方法中**不能**使用this、super，原因是this和super代表当前对象，静态方法不依赖对象。

> :warning:重要的访问限制
>
> 静态方法可以直接访问静态属性和静态方法
>
> 静态方法不能直接访问实例属性和实例方法
>
> 原因是静态方法调用时，可能还没有创建对象，实例成员依赖对象存在。

### 3、如何理解静态属性和静态方法

小区里面业主的房子可以看成一个一个的对象，这些都是属于特定的业主所独有。但是小区里面也有一些公共设施（健身器材、绿地），这些公共设施是属于这个小区的。但是是可以被所有的业主对象所使用的。

非静态的属性和方法（实例属性和方法）必须通过`new`对象访问，而静态的属性和方法是属于类的，在类加载到内存中就可以访问，不需要new对象就可以使用，但是**可以被所有对象所共享**。

例如下面这种访问方式是正确的，但是不推荐这样使用：

```java
public class Student(){
    public static String country;
}

public class test(){
    public static void main () {
        Student stu = new Student;
        stu.country = "中国";
    }
}
```

与静态方法相对的叫**实例方法**或者**非静态方法**。

## 九、工具类

例如 Math、Arrays 等都是工具类，工具类有以下特点：

- 工具类里面的方法都是**静态方法**，方法之间没有关联。

- 工具类的**构造方法私有（Private）**，禁止外界 new 创建对象，只能通过 **类名.方法** 的方式访问

当然也可以自行编写数据类，比如自行编写一个数组的工具类ArraysUtil：

```java
public class ArraysUtil {
    //构造方法私有，禁止New对象
    private ArraysUtil() {
    }

    public int max(int[] array){
        if(array == null || array.length == 0){
            return -1;//长度为0或Null返回-1
        }
        int max = array[0];//初始化最大值为数组第一个元素
        for(int i = 1; i < array.length; i++){
            if(array[i] > max){
                max = array[i];//更新最大值
            }
        }
        return max;//返回最大值
    }

    public int min(int[] array){
        if(array == null || array.length == 0){
            return -1;//长度为0或Null返回-1
        }
        int min = array[0];//初始化最小值为数组第一个元素
        for(int i = 1; i < array.length; i++){
            if(array[i] < min){
                min = array[i];//更新最小值
            }
        }
        return min;//返回最小值
    }

    public int[] bubbleSort(int[] array){
        if(array == null || array.length == 0){
            return null;//长度为0或Null返回null
        }
        int[] sortedArray = array.clone();//克隆原数组
        for(int i = 0; i < sortedArray.length - 1; i++){
            for(int j = 0; j < sortedArray.length - 1 - i; j++){
                if(sortedArray[j] > sortedArray[j + 1]){
                    int temp = sortedArray[j];
                    sortedArray[j] = sortedArray[j + 1];
                    sortedArray[j + 1] = temp;
                }
            }
        }
        return sortedArray;//返回排序后数组
    }
}
```
