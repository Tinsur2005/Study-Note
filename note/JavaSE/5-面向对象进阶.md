## 二、继承

### 1、介绍

让一个类（子类 / 派生类）去复用另一个类（父类 / 超类）的`非私有成员`，实现代码复用，是面向对象三大特征之一，使用关键字`extends`。

父类：`super class`

子类：`sub class`

Java的继承是单继承，即一个类只能直接继承`一个`父类；也可以实现多层继承（A→B→C），`Object`是所有类的父类（这个父类不一定是直接父类）

:warning:Java里面创建（new）子类对象的时候会首先把父类对象创建（new）出来。

按照本节中`一、UML类图`中展示的UML类图所展示的关系编写代码：

```java
public class Person {
    protected int id;
    protected String name;
    protected int age;
    protected String gender;

    public void show() {
        System.out.println("Person.show");
    }
}

public class Student extends Person{
    private String className;

    public void study() {
        System.out.println("Student.study");
    }
}

public class Teacher extends Person{
    private double salary;

    public void teach() {
        System.out.println("Teacher.teach");
    }
}
```

### 2、继承中的this和super

**this：**代表当前类的对象 `this.id=id`

**super：**代表父类对象，在子类中调用父类的属性或方法

**this()：**调用当前类的其他构造方法

**super()：**调用父类构造方法

需要注意的是，在子类中写一个无参构造，当不写任何代码时，默认添加一个`super()`来调用父类的无参构造。

## 三、重载

同一个类中，`方法名`相同，`参数列表`不同，多个方法互为重载（Overload）。

Java通过方法的`参数类型`、`个数`或`顺序`来区分不同的重载方法。

**最典型的重载：**构造方法重载（无参构造方法和有参构造方法，方法名相同，但参数列表不同）

## 四、重写

**重写（Override）**发生在父子类（继承关系）之间
子类定义一个方法，方法签名和父类被继承的方法完全一致，子类覆盖父类方法实现，称为重写，重写方法前要加上`@Override`注解.

满足条件：存在继承（extends）或实现（implements）关系，方法名、参数列表必须和父类一模一样。

>- 子类覆盖父类，必须保证子类权限大于父类权限才可以覆盖，否则会编译失败。
>
>- 如果父类是`private`的方法，子类是默认方法，那么子类是不知道这个方法的，不是覆盖。

```java
class Animal{
    public Animal getObj(){return null;}
}
class Dog extends Animal{
    @Override
    public Dog getObj(){return null;}
}
```

## 五、多态

### 1、介绍

**Java多态（Polymorphism）**是面向对象编程的一项核心概念，它允许对象以`多种形式`出现。

面向对象三大特征：封装、继承、**多态**。

>多态存在的三个必要条件：
>
>- 继承
>
>- 重写
>
>- 父类引用指向子类对象（向上转型）
>
>  ```java
>  Animal animal = new Dog(); // 向上转型，多态写法
>  animal.say(); // 执行子类重写的方法
>  ```

![6dbd8b3d-e467-4f1c-938a-4a2960ffecb3](./../assets/5-%E9%9D%A2%E5%90%91%E5%AF%B9%E8%B1%A1%E8%BF%9B%E9%98%B6/6dbd8b3d-e467-4f1c-938a-4a2960ffecb3.png)

```java
public class Cal {
    protected int num1;
    protected int num2;

    public Cal(int num1, int num2){
        this.num1 = num1;
        this.num2 = num2;
    }

    public int getResult(){
        return 0;
    }

}

public class Add extends Cal{

    public Add(int num1, int num2){
        super(num1, num2);
    }

    @Override
    public int getResult(){
        return num1 + num2;
    }
}

public class Sub extends Cal{
    public Sub(int num1, int num2){
        super(num1, num2);
    }

    @Override
    public int getResult() {
        return num1 - num2;
    }
}

public class Mul extends Cal{
    public Mul(int num1, int num2){
        super(num1, num2);
    }
    @Override
    public int getResult(){
        return num1 * num2;
    }
}

public class Div extends Cal{
    public Div(int num1, int num2) {
        super(num1, num2);
    }
    @Override
    public int getResult() {
        return num1 / num2;
    }
}
```

实现多态：

```java
@Test
public void test2() {
    Cal cal = new Add(3,2);
    System.out.println(cal.getResult());

    cal = new Sub(3,2);
    System.out.println(cal.getResult());
}
```

输出：

```bash
5
1
```

### 2、编译时多态和运行时多态

多态又分为`编译时多态（静态多态）`和`运行时多态（动态多态）`，Java真正意义上的多态是`运行时多态`。

**编译时多态**，又称静态多态，对应的是`方法重载（Overload）`；

**运行时多态**，又称动态多态，对应的是`方法重写（Override）`，依托`向上转型`实现多态。

| 对比项       | 编译时多态（静态多态） | 运行时多态（动态多态） |
| ------------ | ---------------------- | ---------------------- |
| 实现方式     | 方法重载 Overload      | 方法重写 Override      |
| 发生时机     | 编译期绑定             | 运行期绑定             |
| 绑定类型     | 静态绑定 / 早绑定      | 动态绑定 / 晚绑定      |
| 位置         | 同一个类               | 父子继承体系           |
| 是否需要继承 | 不需要                 | 必须要有继承 / 实现    |
| 核心判断依据 | 参数列表               | 对象实际类型           |
